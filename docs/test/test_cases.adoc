== Test Case: TC01 - Erfolgreiche Pilotenregistrierung

=== Beschreibung
Dieser Testfall überprüft, ob die Registrierung eines neuen Piloten korrekt im System erfasst wird. Der Administrator gibt alle erforderlichen Informationen für einen neuen Piloten ein und übermittelt diese Daten.

=== Vorbedingungen
* Der Administrator ist im System eingeloggt.
* Es gibt keine Einschränkungen bezüglich der Anzahl registrierbarer Piloten.

=== Nachbedingungen
* Ein neuer Pilot ist erfolgreich registriert.
* Eine Bestätigungsnachricht über die erfolgreiche Registrierung erscheint im System.

== Test Case: TC02 - Pilotenanmeldung mit gültigen Anmeldeinformationen

=== Beschreibung
In diesem Testfall soll getestet werden, ob ein Pilot sich erfolgreich mit seinen gültigen Anmeldeinformationen einloggen kann.

=== Vorbedingungen
* Der Pilot ist bereits im System registriert.
* Die Webapp ist über einen QR-Code oder einen Direktlink erreichbar.

=== Nachbedingungen
* Der Pilot ist im System angemeldet.
* Der Pilot erhält Zugriff auf für ihn freigegebene Bereiche und Funktionen des Systems.

== Test Case: TC03 - Protokollerstellung durch einen angemeldeten Piloten

=== Beschreibung
Dieser Testfall prüft, ob ein angemeldeter Pilot erfolgreich ein Flugprotokoll anlegen kann. Der Pilot füllt alle erforderlichen und optionalen Felder des Protokolls aus und sendet dieses ab.

=== Vorbedingungen
* Der Pilot ist im System angemeldet.
* Alle erforderlichen Dienste für das Abfragen von Wetterdaten sind verfügbar.

=== Nachbedingungen
* Ein neues Protokoll ist erstellt und im System gespeichert.
* Das Protokoll enthält alle eingegebenen Informationen.

== Test Case: TC04 - Automatische Zuweisung des Flugleiters

=== Beschreibung
In diesem Testfall wird geprüft, wie das System automatisch einen Flugleiter zuweist, wenn mehrere Piloten angemeldet sind. Es wird getestet, ob der dritte angemeldete Pilot automatisch als Flugleiter eingetragen wird.

=== Vorbedingungen
* Mindestens zwei Piloten sind bereits angemeldet.
* Ein weiterer Pilot ist bereit, sich anzumelden.

=== Nachbedingungen
* Der dritte angemeldete Pilot wird automatisch als Flugleiter festgelegt.
* Der Flugleiter erhält spezifische Benachrichtigungen oder Zugriffe, die seine Rolle betreffen.


== Test Case : TC05 - Sicherheitsüberprüfungen

=== Beschreibung

In diesem Test-Case wird die nichtfunktionale Anforderung "Legal" auf Erfüllung getestet. Die vollständige Website (außer Login-Page, Datenschutzerklärung, Impressum), die personalisierten Daten sowie Dokumente der Datenbank und des Webservers dürfen nur nach der Eingabe von Passwörtern lesbar, veränderbar oder löschbar sein.

=== Vorbedingungen

- Der Akteur ist nicht angemeldet

=== Testablauf

- Die Website sollte nur die oben genannten Seiten anzeigen
- Der Versuch weitere Seiten aufzurufen sollte ohne Anmeldung nicht möglich sein

=== Nachbedingungen

- Es sollte keine Änderung/Anzeige verbindlicher Daten möglich sein
- Nur die genannten Teile der Website sollten erreichbar sein
- Die Funktionen der Website, welche nach der Anmeldung zur Verfügung stehen, sollten ohne Anmeldung nicht erreichbar sein
